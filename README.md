# Dartmouth Code Writers Guild: Reading Group

![Meetings](https://img.shields.io/badge/meetings-monthly-blue)
![Duration](https://img.shields.io/badge/duration-45min-green)
![Format](https://img.shields.io/badge/format-hybrid-purple)

A monthly forum for Dartmouth members who write code to discuss software engineering practices, tools, and ideas through bite-sized content.

## ðŸŽ¯ Purpose

This reading group brings together software engineers, research software engineers, and anyone who writes code at Dartmouth to:

- **Learn** about industry practices and emerging ideas in software engineering
- **Discuss** how these concepts apply to our work at Dartmouth
- **Share** experiences and perspectives from our diverse projects
- **Improve** our craft through collective learning

We focus on practical, actionable content that can influence how we build software.

## ðŸ“… Format

- **Frequency:** Monthly (typically 2nd Thursday, 2:00-2:45 PM ET)
- **Duration:** 45 minutes
- **Location:** Hybrid (room TBD + video conference link in calendar invite)
- **Materials:** Usually short-form content (~15-30 min reading/watching), occasionally longer
  - Blog posts
  - Conference talks
  - Style guides
  - Documentation
  - Short articles or papers

### Typical Session Flow

```
0:00-0:05  Welcome & recap
0:05-0:10  Brief summary of material (facilitator)
0:10-0:40  Facilitated discussion
0:40-0:45  Takeaways & next topic selection
```

## ðŸš€ Getting Started

### For Participants

1. **Join the group:** Watch this repository to get notifications
2. **Review materials:** Check the [upcoming session](#-upcoming-sessions) 1-2 weeks before
3. **Engage:** Add thoughts to the discussion thread before the meeting
4. **Attend:** Join us live or catch up via meeting notes afterward
5. **Contribute:** Suggest topics via [GitHub Issues](../../issues/new?template=topic-suggestion.md)

### For First-Timers

No preparation required! You can:
- Come to any session (no need to attend all)
- Participate as much or as little as you want
- Skip the reading and just listen
- Join remotely or in person

We're here to learn together, not quiz each other.

## ðŸ¤ Discussion Guidelines

- **Be curious:** Ask questions and assume good intent
- **Be inclusive:** Make space for all voices
- **Be respectful:** Disagree with ideas, not people
- **Be practical:** Connect concepts to real work
- **Be open:** "Yes, and..." over "No, but..."
